## Звіт про RAG та підходи до retrieval

### Що таке RAG?

**Retrieval-Augmented Generation (RAG)** — це техніка, яка поєднує потужність великих мовних моделей з зовнішнім отриманням знань. Замість того, щоб покладатися виключно на знання, закодовані в параметрах моделі під час навчання, системи RAG можуть отримувати та інтегрувати актуальну, специфічну для домену інформацію під час інференції.

#### Основні компоненти RAG:
1. **Завантаження документів**  
   - Завантаження документів (PDF, TXT, HTML тощо)  
   - Розбиття тексту на частини (chunking)  
   - Генерація векторів (embedding generation)  
   - Зберігання векторів (FAISS, Pinecone, Chroma тощо)

2. **Методи отримання інформації**  
   - **Семантичний пошук**: Використовує щільні векторні вбудування для знаходження семантично подібних документів.  
     - **Переваги**: Розуміє значення, обробляє синоніми.  
     - **Недоліки**: Може пропустити точні збіги ключових слів.  
   - **Лексичний пошук (BM25)**: Традиційний пошук на основі ключових слів з використанням частоти термінів.  
     - **Переваги**: Швидкий, хороший для точних збігів.  
     - **Недоліки**: Не розуміє семантики.  
   - **Гібридний пошук**: Поєднує семантичний та лексичний пошук для досягнення найкращих результатів.

3. **Перенумерація (reranking)**: Моделі крос-кодування, які точніше оцінюють пари запит-документ.

#### Кращі практики:
- Розмір частин має значення — занадто маленькі втрачають контекст, занадто великі — шумні.
- Перекриття між частинами зберігає контекст на межах.
- Фільтрація метаданих покращує релевантність.
- Перенумерація значно покращує якість.
- Гібридний пошук перевершує окремі методи.

#### Загальні виклики:
- Вибір стратегії розбиття.
- Обробка мультимодальних документів.
- Балансування швидкості та точності.
- Управління великими колекціями документів.
- Оновлення вбудувань.

### Підходи до retrieval

1. **Семантичний пошук**  
   - Використання векторних вбудувань для знаходження документів, які мають схоже значення.

2. **Лексичний пошук**  
   - Використання традиційних методів на основі частоти термінів для знаходження документів.

3. **Гібридний пошук**  
   - Поєднання семантичного та лексичного пошуку для досягнення кращих результатів.

4. **Перенумерація**  
   - Використання моделей, які оцінюють пари запит-документ для покращення точності результатів.

### Висновок

RAG є потужним підходом, що дозволяє інтегрувати зовнішні знання в процес генерації тексту, а різноманітні методи отримання інформації забезпечують гнучкість та точність у пошуку релевантних документів.
