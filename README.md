# 🔬 Multi-Agent Research System (MCP + ACP)

Автономна система для глибокого дослідження тем. Це не просто чат-бот, а команда агентів, яка вміє самостійно планувати пошук, аналізувати локальні документи та верифікувати інформацію в мережі.

### 🤖 Як це працює
Процес повністю автоматизований завдяки координації ролей:
1.  **Planner** — аналізує запит, створює стратегію пошуку та структуру звіту.
2.  **Researcher** — «робоча конячка». Шукає дані в локальній базі (RAG) та через DuckDuckGo, витягує контент із веб-сторінок.
3.  **Critic** — перевіряє знахідки на актуальність та повноту. Якщо бачить прогалини — відправляє на допрацювання.
4.  **Supervisor** — керує циклом, викликає ACP-сервіси та запитує ваше підтвердження перед збереженням звіту.

### 🛠 Технологічний стек
* **Основа:** LangGraph (цикли агентів), LangChain.
* **LLM:** GPT-4o-mini.
* **RAG:** FAISS + BM25 (гібридний пошук: семантика + точні збіги).
* **Інфраструктура:** MCP (Model Context Protocol) для інструментів пошуку та запису файлів.

### 🚀 Швидкий старт
1.  **Встановіть залежності:** `pip install -r requirements.txt`.
2.  **Налаштуйте ключі:** Створіть `.env` та додайте `OPENAI_API_KEY`.
3.  **Індексуйте дані:** Покладіть PDF/TXT у папку `data/` та запустіть `python ingest.py`.
4.  **Запустіть сервери (у різних терміналах):**
    * `python mcp_servers/search_mcp.py` (порт 8901)
    * `python mcp_servers/report_mcp.py` (порт 8902)
    * `python acp_server.py` (порт 8903)
5.  **Запустіть головний інтерфейс:** `python main.py`.

### 💡 Що тут цікавого
* **Human-in-the-loop:** Система виводить прев'ю звіту і чекає на вашу команду (`approve`, `edit` або `reject`).
* **Якість даних:** Використання реранкінгу та гібридного пошуку мінімізує «шум» у відповідях.
* **Цикл доопрацювання:** Якщо Critic незадоволений, Researcher робить до 2-х додаткових ітерацій пошуку.

*Фінальні звіти зберігаються у папку `/output` у форматі Markdown.*
