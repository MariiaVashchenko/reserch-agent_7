# Мультиагентна Дослідницька Система (LangGraph)

Система на базі ШІ для проведення глибоких досліджень з використанням архітектури LangGraph, гібридного пошуку (RAG) та циклу автоматичної критики.

## 🏗 Архітектура системи

Проєкт реалізує патерн **"Supervisor" (Оркестратор)**, який координує роботу трьох спеціалізованих агентів:

1.  **Planner Agent**: Аналізує запит користувача, робить попередній пошук та формує детальний план дослідження.
2.  **Research Agent**: Виконує пошук у локальній базі знань (PDF, TXT) та в інтернеті, агрегує знайдені факти.
3.  **Critic Agent**: Оцінює звіт за критеріями актуальності, повноти та структури. Може відправити звіт на доопрацювання.
4.  **Supervisor**: Керує переходами між агентами, зберігає стан (пам'ять) та ініціює збереження фінального звіту.

## 🚀 Основні можливості

- **Hybrid Retrieval**: Комбінація семантичного пошуку (FAISS + OpenAI embeddings) та лексичного пошуку (BM25).
- **Reranking**: Використання Cross-Encoder моделей для фільтрації шуму та підвищення релевантності знайдених даних.
- **HITL (Human-in-the-Loop)**: Система зупиняється перед збереженням звіту, дозволяючи користувачу підтвердити, відредагувати або відхилити результат.
- **Self-Correction**: Автоматичний цикл покращення контенту (до 2 раундів ревізії між Дослідником та Критиком).
- **Structured Output**: Використання Pydantic-моделей для гарантії чіткої структури відповідей агентів.

## 🛠 Встановлення

1. **Клонуйте репозиторій:**
   ```bash

   cd homework-lesson-8
   ```

2. **Створіть та активуйте віртуальне середовище:**
   ```bash

   # Windows:
   venv\Scripts\activate

   source venv/bin/activate
   ```

3. **Встановіть залежності:**
   ```bash
   pip install -r requirements.txt
   ```

4. **Налаштуйте змінні оточення:**

   ```env
   OPENAI_API_KEY=ваш_ключ_openai
   ```

## 📂 Структура проєкту

- `main.py`: Точка входу, інтерфейс REPL з підтримкою переривань (HITL).
- `supervisor.py`: Головний оркестратор та логіка координації агентів.
- `agents/`: Модулі окремих агентів (планувальник, дослідник, критик).
- `retriever.py`: Система гібридного пошуку та реранкінгу.
- `ingest.py`: Скрипт для індексації документів з папки `data/`.
- `schemas.py`: Pydantic-схеми для структурованих даних.
- `config.py`: Системні промпти, налаштування моделей та ліміти.
- `tools.py`: Інструменти пошуку та роботи з файлами.

## 📝 Використання

### 1. Підготовка бази знань
Завантажте ваші документи (PDF, MD, TXT) у папку `data/` та запустіть індексацію:
```bash
python ingest.py
```

### 2. Запуск системи
Запустіть основний цикл програми:
```bash
python main.py
```

### 3. Керування в консолі
- Просто введіть тему для дослідження.
- `new` — розпочати нову сесію (очистити пам'ять агента).
- `status` — перевірити стан завантаженої бази знань.
- `exit` — вихід з програми.


## ⚙️ Налаштування (config.py)

Ви можете змінити параметри системи в `config.py`:
- `MODEL_NAME`: Модель за замовчуванням (напр. `gpt-4o-mini`).
- `MAX_REVISION_ROUNDS`: Максимальна кількість циклів "дослідження-критика".
- `USE_RERANKER`: Увімкнення/вимкнення Cross-Encoder для економії ресурсів.
